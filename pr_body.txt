## Summary

The `_run_with_spawn()` method had a fallback block (lines 219-230) that triggered whenever spawn returned a FAIL status. When triggered, it would re-run the node via `_run_with_tool_loop()` without the child session's thread history.

This silently bypasses intentional goal_gate failures. For example, a `quality_eval` node correctly returning `{"status":"fail"}` would be converted to success by re-running the node fresh, with no log warning.

Infrastructure spawn failures are already handled separately via try/except in `_run_with_spawn` (exception → early return FAIL). The fallback block only fires on intentional FAIL outcomes, making it harmful.

## The Fix

- **Remove** the fallback block (lines 219-230)
- **Replace** with clarifying comment explaining why the fallback is incorrect
- **Keep** the `elif self._provider is not None:` branch for direct tool-loop execution when spawn is unavailable
